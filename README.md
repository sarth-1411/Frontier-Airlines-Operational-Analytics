## 📌 Overview

This project transforms **four raw BTS Form 41 `.xls` filings** covering Frontier Airlines (1995–2020) into a clean, analysis-ready data model. It demonstrates end-to-end airline analytics — from messy regulatory source files through ETL engineering to Power BI dashboards and interactive visualization.

**What makes this non-trivial:** The raw files store data in pivot-table format (years as columns, 500+ metrics as rows) with dual-header structures, inconsistent column offsets across files, and early-year zero-fills requiring careful handling. The ETL pipeline normalizes all of this into a proper star schema.

### Key Finding
> Frontier's 2014–2016 ULCC pivot shows a **rare simultaneous productivity increase and cost reduction** — ASMs per employee doubled (3.2M → 5.6M) while ex-fuel CASM declined. This structural shift, driven by fleet densification and stage length optimization, is the core story in the data.

## 📐 ETL: What the Script Does

The raw files present several real-world data engineering challenges:

| Challenge | How It's Handled |
|---|---|
| **Pivot layout** | Years stored as columns — transposed to normalized row-per-year format |
| **Dual headers** | Col A = section label, Col B = metric label — both required to identify each series |
| **Column offset mismatch** | Employee file years start at col 1, all others at col 2 — separate `extract_emp_row()` function |
| **Early-year zero-fills** | 1995–1997 had minimal BTS reporting — treated as `None`, not `0` |
| **500+ metrics** | Mapped to meaningful column names with units documented in code |
| **Derived KPIs** | Ancillary revenue, RASM-CASM spread, ASM/gallon fuel efficiency, revenue/employee |
| **Era classification** | 5 strategic eras added for Power BI slicer segmentation |

## 🔍 Key Analytical Findings

**1. ULCC Productivity Inflection (2014–2016)**
ASMs per employee doubled from 3.2M to 5.6M over two years while ex-fuel CASM simultaneously declined — a structural efficiency gain, not a demand tailwind. Driven by outsourcing, automation, and fleet right-sizing.

**2. Fleet Densification as Cost Strategy**
Seats per departure grew from 110 (1995) to 193 (2019) — a 75% increase. Each additional seat on the same flight reduces per-seat fixed cost with zero incremental block hour cost.

**3. The Yield-Load Factor Trade-off**
Yield (revenue per RPM) fell 63% from 1995 to 2019 (15.4¢ → 5.7¢) while load factor rose 37 percentage points. The ULCC model deliberately sacrifices unit revenue for volume — total revenue grew 45× over the same period.

**4. Fuel Crisis Resilience (2008)**
When fuel hit $3.33/gallon, CASM spiked to 7.7¢ — but RASM reached 9.9¢, maintaining a positive spread. Ex-fuel cost discipline created the buffer that kept Frontier profitable through the commodity shock.

**5. Stage Length & Network Depth**
Average stage length grew 135% (447 → 1,050 miles). Longer flights improve aircraft utilization and reduce per-mile costs — every scheduling decision that extends stage length improves the unit cost structure.

## 🛠️ Power BI Setup

See [`PowerBI_Setup_Guide.md`](PowerBI_Setup_Guide.md) for full instructions. Quick summary:

1. **Load** all 8 CSVs from `clean_data/` into Power BI Desktop
2. **Create relationships** in Model view (7 relationships, all Many-to-One)
3. **Add measures table** — copy 30+ DAX measures from the guide
4. **Build 6 dashboard pages**: Executive Summary · Unit Economics · Workforce · Fleet · Geography · Fuel

Sample DAX measures included:
```dax
[RASM-CASM Spread]  = AVERAGE(fact_annual_operations[RASM_cents])
                    - AVERAGE(fact_annual_operations[CASM_cents])

[Productivity Index] = DIVIDE(
    [ASMs per Employee (000)],
    CALCULATE([ASMs per Employee (000)], dim_date[Year] = 2010)
)

[Revenue YoY %] = VAR cur = [Total Revenue $M]
                  VAR prior = CALCULATE([Total Revenue $M],
                              dim_date[Year] = MAX(dim_date[Year]) - 1)
                  RETURN DIVIDE(cur - prior, prior)
```

## 📦 Data Source

All data sourced from the **U.S. Bureau of Transportation Statistics (BTS), Form 41**:

| Source File | BTS Schedule | Content |
|---|---|---|
| Employee Data | P10, P6, P12, P52 | Headcount, salaries, operating expenses, P&L |
| Aircraft Actuals | P5.2 / T2 | Operating expense by fleet type |
| Traffic / Capacity | T2 | ASM, RPM, revenue, RASM/CASM/yield by region |
| Cost Per Block Hour | P5.2 | CPBH by expense category and fleet type |

> Data covers **Frontier Airlines, Inc.** — a subsidiary of Frontier Group Holdings, Inc. (NASDAQ: ULCC), headquartered in Denver, Colorado.
